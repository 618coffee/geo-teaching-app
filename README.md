# Geo Teaching App

面向高中地理“工业区位选择”场景的前端教学应用 MVP。

当前版本将学生端、教师端、运营端放在同一个 React 单页应用中，用示例数据先跑通页面结构、路由切换、教学流程和分析展示，方便继续验证交互方案、课堂流程和后续平台化扩展。

## 项目目标

- 验证高中地理“工业区位选择”教学场景的完整前端链路
- 在没有后端服务的前提下，用示例数据模拟任务、选址、分析报告和管理页面
- 为后续接入真实 API、鉴权、题库配置和 AI 能力保留清晰的页面骨架

## 当前包含的功能

### 首页认证

- 支持手机号或邮箱注册
- 注册时必须输入验证码，演示环境会直接在页面展示 6 位验证码
- 支持密码登录和验证码登录
- 登录态持久化到浏览器本地存储
- 学生端、教师端、运营端均已接入路由级角色访问控制

### 学生端

- 任务列表：查看教学任务、班级、地图模式、截止时间和完成率
- 区位决策：在地图热区中选择工厂位置，填写班级码，切换工厂类型
- 分析报告：基于工厂类型偏好和区域属性生成示例分析结果
- 对比报告：比较候选区域方案
- 反向推荐：从产业偏好反推更合适的区位方案

### 教师端

- 任务管理：查看当前教学任务和进度
- 发布任务：配置教学任务、班级和区位考察重点
- 作业批阅：查看学生提交记录和批阅状态
- 学情分析：通过图表查看区域分布、完成率和典型问题
- 课堂演示：展示公开课或课堂演示记录

### 运营端

- 模块管理：维护教学模块、案例库和模块状态
- 人员管理：展示平台侧人员协作信息
- Prompt 配置：为后续 AI 生成和分析流程预留配置入口

## 快速开始

### 运行环境

- Node.js 18 或更高版本
- npm 9 或更高版本
- Java 21 或更高版本（用于 `server/` Spring Boot 后端）

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

默认会启动在 `http://localhost:5173`。

启动后可以从首页进入不同角色页面：

- 学生端：`/student/tasks`
- 教师端：`/teacher/tasks`
- 运营端：`/admin/modules`

### 构建生产包

```bash
npm run build
```

### 启动后端开发环境

```bash
npm run dev:server
```

默认会启动在 `http://localhost:4000`。

后端为 Spring Boot 骨架，也可以单独进入 `server/` 目录执行：

```bash
cd server
sh ./mvnw spring-boot:run
```

### 本地预览构建结果

```bash
npm run preview
```

## Demo 数据说明

- 当前所有页面数据都来自 `src/data/demoData.ts`
- 学生端的班级码、工厂类型和已选区域会保存在浏览器 `localStorage`
- 本地持久化 key 为 `geo-teaching-demo-state`
- 选址分析由示例规则逻辑生成，本质是“工厂类型权重 × 区域属性分数”的前端演示逻辑

## 当前范围与限制

- 首页认证当前仍默认走前端演示模式，账号、会话和验证码暂存于浏览器 `localStorage`
- 仓库已提供独立 Spring Boot 后端骨架，但前端尚未切到真实 API
- 没有接入真实短信、邮件、数据库和服务端会话
- 自动化测试和部署配置仍需继续补充
- 页面重点是教学流程演示，不代表最终产品交互或视觉定稿

## 国内后端选型建议

如果这个应用的主要用户面向中国大陆，默认推荐优先考虑阿里云，原因是：

- 短信服务、邮件推送、RDS、Redis、对象存储、CDN、WAF 和备案配套都比较完整
- 国内教育、内容平台和中后台系统的实践案例更多，资料和第三方生态也更成熟
- 如果后续要做高并发验证码、静态资源分发、日志审计和多地域容灾，落地路径相对清晰

建议的第一版后端组合：

- 计算层：阿里云 SAE 或 ECS
- API 层：Spring Boot / NestJS / Go 任一常规 Web 服务
- 数据库：阿里云 RDS MySQL
- 缓存：阿里云 Redis，用于验证码、登录限流、短时会话缓存
- 短信：阿里云短信服务
- 邮件：阿里云邮件推送
- 文件：阿里云 OSS

如果你的产品后续会深度接入微信生态，可以把腾讯云作为第二候选；如果更偏政教项目、对本地化交付和国产化要求更高，可以再评估华为云。

从当前这个项目的需求看，首选结论可以先定为：

- 默认选阿里云
- 如果强绑定微信生态，再看腾讯云
- 如果是高校/政教采购驱动，再看华为云

更完整的接口设计、部署拓扑和环境变量说明见：

- `docs/aliyun-auth-backend-plan.md`

## 后端脚手架

- 当前仓库已新增 `server/` 目录，提供一个基于 Spring Boot 的认证后端骨架
- 已实现健康检查、验证码发送、注册、密码登录、验证码登录、获取当前用户、退出登录等基础接口
- 默认使用本地 JSON 文件保存开发数据，并通过 `mock` 通知通道模拟短信/邮件发送
- 生产环境建议将用户数据迁移到阿里云 RDS MySQL，将验证码和限流迁移到阿里云 Redis，并接入阿里云短信与邮件推送
