# Geo Teaching App — Frontend

面向高中地理"工业区位选择"场景的 React 单页教学应用。

当前版本将学生端、教师端、运营端放在同一个 React 单页应用中，用示例数据先跑通页面结构、路由切换、教学流程和分析展示。

## 功能

### 首页认证

- 支持手机号或邮箱注册
- 注册时必须输入验证码，演示环境会直接在页面展示 6 位验证码
- 支持密码登录和验证码登录
- 登录态持久化到浏览器本地存储
- 学生端、教师端、运营端均已接入路由级角色访问控制

### 学生端

- 任务列表：查看教学任务、班级、地图模式、截止时间和完成率
- 区位决策：在地图热区中选择工厂位置，填写班级码，切换工厂类型
- 分析报告：基于工厂类型偏好和区域属性生成示例分析结果
- 对比报告：比较候选区域方案
- 反向推荐：从产业偏好反推更合适的区位方案

### 教师端

- 任务管理：查看当前教学任务和进度
- 发布任务：配置教学任务、班级和区位考察重点
- 作业批阅：查看学生提交记录和批阅状态
- 学情分析：通过图表查看区域分布、完成率和典型问题
- 课堂演示：展示公开课或课堂演示记录

### 运营端

- 模块管理：维护教学模块、案例库和模块状态
- 人员管理：展示平台侧人员协作信息
- Prompt 配置：为后续 AI 生成和分析流程预留配置入口

## 技术栈

- React 18
- TypeScript
- Vite 5
- React Router 6
- ECharts

## 项目结构

```text
src/
├── auth/            # 认证上下文
├── components/      # 图表、地图面板等复用组件
├── data/            # 示例数据与 demo 状态
├── layouts/         # 应用壳和导航布局
├── pages/           # 学生端 / 教师端 / 运营端页面
├── App.tsx          # 路由入口
├── main.tsx         # 应用挂载与路由初始化
└── styles.css       # 全局样式
```

## 快速开始

### 运行环境

- Node.js 18 或更高版本
- npm 9 或更高版本

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

默认会启动在 `http://localhost:5173`。

启动后可以从首页进入不同角色页面：

- 学生端：`/student/tasks`
- 教师端：`/teacher/tasks`
- 运营端：`/admin/modules`

### 构建生产包

```bash
npm run build
```

### 本地预览构建结果

```bash
npm run preview
```

## Demo 数据说明

- 当前所有页面数据都来自 `src/data/demoData.ts`
- 学生端的班级码、工厂类型和已选区域会保存在浏览器 `localStorage`
- 本地持久化 key 为 `geo-teaching-demo-state`
- 选址分析由示例规则逻辑生成，本质是"工厂类型权重 × 区域属性分数"的前端演示逻辑

## 当前范围与限制

- 首页认证当前仍默认走前端演示模式，账号、会话和验证码暂存于浏览器 `localStorage`
- 前端尚未切到真实后端 API
- 自动化测试和部署配置仍需继续补充
- 页面重点是教学流程演示，不代表最终产品交互或视觉定稿
